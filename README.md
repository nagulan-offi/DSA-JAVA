## How I approach every problem

```
1. Read the problem — understand input, output, constraints
2. Think for 20 minutes without looking at any hint
3. Write the brute force first — then optimise
4. Code the solution in Java
5. Analyse time and space complexity before submitting
6. Push to this repo with approach in commit message
```

---

## Commit message format

```
Add: Problem Name (#number) — approach used
```

Examples:
```
Add: Trapping Rain Water (#42) — precompute lmax rmax O(n)
Add: Maximum Subarray (#53) — Kadane's Algorithm O(n)
```
